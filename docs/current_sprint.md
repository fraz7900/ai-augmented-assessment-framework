Current sprint: Sprint 14 — PCI DSS (Section-level statement-only) + cross-framework equivalence + a real bug fix
Status: Complete. This sprint covers the final named framework-breadth item from `PROJECT_CHARTER.md` Section 13. PCI DSS: confirmed directly that PCI DSS v4.0.1 is copyrighted, all-rights-reserved content (the source PDF's own page 1 states this explicitly) despite being freely downloadable at no cost — the same situation as ISO 27001/SOC 2. Applied that precedent directly. A second, independent scope decision was also required: PCI DSS is uniquely three hierarchy levels deep (Requirement → Section → Defined Approach Requirement, ~205 leaf items — no other framework here has this depth), so this transcription stops at the Section level (63 of 63 real, verified Section statements across 12 Requirements, `framework_mapping/pci_dss_v4.yaml`) rather than the disproportionately larger leaf level, with the leaf level named as real, disclosed future work. Cross-framework equivalence: 80 of 141 NERC CIP practices matched against PCI DSS, disclosed as statement-level, not full-text — bringing NERC CIP's total reviewed coverage (across C2M2, ISO 27001, CIS Controls, SOC 2, and PCI DSS) to 118 of 141 practices. PCI DSS has no dedicated recovery-plan Section, so CIP-009 barely matched; CIP-014 found zero matches, the sharpest single-standard gap of any pairing reviewed so far. A real, previously-latent bug was found and fixed in the same pass (not just disclosed as a limitation): `services/framework_loader.py`'s cross-framework practice-text index was keyed by bare `practice_id` alone, so CIS Controls' Safeguard "5.1" and PCI DSS's Section "5.1" (an identical ID string) silently collided, corrupting CIP-007-5.3's CIS Controls equivalent to show PCI DSS's name and text instead — caught by a pre-existing regression test failing once both frameworks were loaded together, fixed by re-keying the index to `(framework_name, practice_id)` and having the merge logic resolve via each equivalence entry's own framework fields. This protects every framework in the project against the same latent risk going forward, not just PCI DSS. 213 backend tests passing (4 new/updated for PCI DSS, 4 fixed for coverage-count changes, the collision-bug regression test, and a stale "PCI DSS = unknown framework" test collision the SOC 2 work had itself introduced). Verified live end to end throughout.
