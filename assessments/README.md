# assessments/

Persisted state for in-progress and completed assessments (which practices have been mapped, which evidence was accepted/rejected, current maturity scores). Structured as versioned JSON per assessment so an assessment's history is itself an audit trail, satisfying the "traceable link from score to evidence" requirement in `PROJECT_CHARTER.md` Section 5 (Internal Audit stakeholder need).

Gitignored at runtime — only this README is tracked. Introduced in Sprint 2 (Assessment Engine).
